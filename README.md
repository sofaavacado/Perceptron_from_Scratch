# Logistic Regression from Scratch

## Общая информация

В этом проекте реализована бинарная классификация на Python с нуля: от генерации и нормализации данных до обучения модели и оценки качества на тестовой выборке.

Проект демонстрирует базовую реализацию линейной модели классификации с использованием:
- сигмоидной функции активации,
- функции потерь cross-entropy,
- градиентного спуска,
- L2-регуляризации,
- визуализации динамики обучения.

Основная цель — показать понимание математической логики обучения модели, а не только использование готовых библиотек.

---

## Ключевые возможности

- Генерация синтетического датасета для задачи бинарной классификации
- Нормализация признаков
- Реализация сигмоиды с защитой от численной нестабильности
- Реализация функции потерь cross-entropy
- Обучение модели градиентным спуском
- Использование L2-регуляризации
- Оценка accuracy на обучающей и тестовой выборках
- Визуализация изменения loss и accuracy по эпохам

---

## Что демонстрирует проект

- Понимание основ бинарной классификации
- Умение реализовывать ML-модель без готовых high-level библиотек
- Знание градиентного спуска и обновления параметров

---

## Архитектура проекта

### Основные этапы

1. **Генерация данных**  
   Создаются синтетические признаки и целевая переменная для задачи бинарной классификации.

2. **Нормализация**  
   Все признаки приводятся к сопоставимому масштабу.

3. **Обучение модели**  
   Параметры модели обновляются с помощью градиентного спуска.

4. **Оценка качества**  
   Рассчитывается accuracy на тестовой выборке.

5. **Визуализация обучения**  
   Строятся графики loss и accuracy по эпохам.

---

## Реализованные компоненты

- **generate_data** — генерация и нормализация данных  
- **sigmoid** — сигмоидная функция активации  
- **cross_entropy_loss** — функция потерь  
- **train_perceptron** — обучение модели  
- **accuracy** — расчёт точности классификации  
- **plot_training_history** — построение графиков обучения  

---

## Метрики и визуализация

Во время обучения отслеживаются:

- **Loss** — значение функции потерь
- **Accuracy** — доля правильных предсказаний

После обучения:
- оценивается качество на тестовой выборке,
- строятся графики изменения loss и accuracy.

Это позволяет проследить, как модель обучается и насколько стабильно улучшается её качество.
